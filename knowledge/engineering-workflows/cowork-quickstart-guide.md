## Key Takeaways
### 1. Skill vs. Connector vs. Plugin Hierarchy
Skills = encoded recipes for specific tasks. Connectors = interfaces with external tools (Gmail, Slack, Stripe, HubSpot). Plugins = bundled skills + connectors (complete kitchen, not just recipes).

### 2. Minimum Viable Setup
Requires Pro account ($17/mo minimum). Create a dedicated folder for Cowork to operate within — never give full desktop access. Sandbox from the start.

### 3. Navigation Quick Tour
Key UI sections: New Task (start work), Search (find old tasks), Scheduled Tasks (recurring), Ideas (quick-start suggestions), Customize (settings). Ideas tab is goldmine for beginners.

### 4. Parallel Task Execution
Start multiple tasks simultaneously without blocking. Perfect for exploratory workflows where sub-tasks run independently then roll up to a final step.

### 5. Built-In Skill Creator
Say "create with Claude" and Cowork guides you through skill building. No coding required. Lower barrier than manual `.md` file authoring.

### 6. Scheduled Tasks for Automation
Daily email summaries, weekly meeting recaps, recurring reports. Set once, run forever. High ROI for repetitive information gathering.

## Relevance to Kvalt

- **Cowork Skills as Development Workflow:** Kvalt's `.skills/` architecture maps directly to Cowork's skill model. Each skill can be extracted and run as a standalone Cowork task — e.g., "run figma-to-kvalt on this file" becomes a scheduled Cowork job.

- **Connector Integration Roadmap:** Kvalt could benefit from Cowork connectors for Figma (sync design changes), GitHub (auto-update component docs), and Slack (team notifications on token changes).

- **Dedicated Workspace Pattern:** Kvalt maintains a documentation-heavy codebase. A Cowork task that periodically checks `docs/` for staleness and flags maintainers is a natural fit — aligns with "Ideas" tab suggestions.

- **Skill Creator for Custom Workflows:** New team members onboarding to Kvalt could use Cowork's built-in skill creator to encode their first contribution workflow — no barrier to automation.
