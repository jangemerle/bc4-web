# The Vibe Coding Trust Gap: Why Technical Knowledge Still Matters

**Source:** Kyle Cook / Web Dev Simplified (YouTube) — "Vibe Coders Will Never Succeed"
**Date:** 2026-04-03
**Added:** 2026-04-04

## Key Takeaways

- **The translator analogy:** If you use Google Translate without knowing the language, you can't verify correctness. Same with AI-generated code. Fine for a casual chat, catastrophic for banking software.
- **Two tiers of AI-built apps:**
  - **Bespoke personal tools:** AI-coded apps for yourself/small group. Quality doesn't matter much. This is where vibe coding shines.
  - **Production/commercial apps:** Need security, scalability, maintainability. Require technical oversight regardless of who/what wrote the code.
- **AI as IDE evolution:** AI is the next step in the same progression as IDEs → autocomplete → linting → AI. Tools for faster output, but the developer's brain is still the product.
- **Security is the breaking point:** Multiple news articles about vibe-coded apps with security breaches. The AI wasn't told to handle security, or did it incorrectly, and the builder couldn't verify either way.
- **Code review as the new core skill:** If AI writes the code, reviewing it becomes the most important developer skill.

## The Scaling Problem

> "As applications grow in scale, it becomes much harder to make sure your code is well written and works across all different aspects. This is where real developers — people with problem-solving skills and technical knowledge — are needed."

Even at 0.01% error rate, AI will mess up. Someone needs to catch it.

## Relevance to Kvalt

- **Faces reduce the trust gap for design:** A vibe coder can't verify whether their AI-generated UI follows good design principles. A face encodes those decisions upfront — the equivalent of a senior designer reviewing every component.
- **check-tokens skill = automated code review for design:** What Kyle advocates (code review as core skill) maps to what check-tokens does — automated verification that the AI's output matches the spec.
- **"Bespoke → production" is Kvalt's market:** People building personal tools who want to level up to production quality. Faces make that jump possible without hiring a design team.
- **Counter-narrative for marketing:** Don't sell "vibe code your design." Sell "production-quality design decisions, even when AI writes the code."
