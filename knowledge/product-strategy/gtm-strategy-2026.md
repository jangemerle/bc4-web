## Distribution channels ranked by relevance to Kvalt

### Tier 1: AI-native distribution (unique to Kvalt's positioning)
- **CLAUDE.md / .cursorrules configs** — Faces distributed as config files AI tools consume. No marketplace exists for design-oriented configs yet. Get into cursor.directory, awesome-cursorrules, aitmpl.com.
- **MCP server** — Live design context injection. Cursor, Claude Code, and other tools connect to Kvalt MCP for active face switching. Harder to build but more defensible.
- **CLI** — `npx @kvalt/faces add fintech` matches shadcn mental model. Source code ownership.

### Tier 2: Product-as-marketing (the Linear model)
- **"Same app, different face" interactive demo** — Toggle between faces on the same SaaS dashboard. This IS the pitch. No words needed.
- **Visual demos** — 15–30 second face-swap videos. Inherently shareable. One exceptional demo per week > seven forgettable posts.
- **GitHub free face pack** — 3–5 faces, copy-paste model. Top-of-funnel for GitHub stars and awesome-list inclusion.

### Tier 3: Community and search (compounding assets)
- **YouTube tutorials** — "I built a SaaS in 30 min with Claude Code, then made it look premium in 5 min with Kvalt." Search-optimized, compounds over time.
- **Reddit value posts** — r/webdev (2.4M), r/programming (6.6M). Authentic problem-solving posts, not promotional.
- **Community embedding** — Be helpful in Cursor, Lovable, Bolt Discord servers. Answer "how do I make this look better" questions.

### Tier 4: Traditional but effective
- **ProductHunt launch** — When product is ready. Frame as "design personality for AI-built products."
- **Indie Hackers** — Build-in-public for founder persona. 23% conversion rate per engaged post.
- **Newsletter** — Owned audience. Build before launch if possible.

## Pricing model (validated by market data)

- **Free:** 3–5 faces + base components (top-of-funnel)
- **Pro:** $149–249 one-time (15+ faces + premium components + Figma kit)
- **Starter:** $49–79 (for vibe coder persona, 5 faces)
- **No subscription** — market has subscription fatigue for component libraries
- **PPP pricing** for EU/CEE markets (40–50% discount)

## Anti-patterns to avoid

1. Don't build distribution AI can bypass (Tailwind's docs/SEO model died)
2. Don't follow generic "post 5x daily on Twitter" advice (marketing AI slop)
3. Don't try to replace shadcn (sit on top of it)
4. Don't position as "designer tool that bypasses developers" (developer resistance is real)
5. Don't do subscriptions for component/config products

## Case studies that inform our approach

| Company | Strategy | Result | Lesson for Kvalt |
|---------|----------|--------|-----------------|
| Linear | Product quality as marketing, $35K total spend | $1.25B valuation | Every screenshot should be share-worthy |
| Aceternity UI | Twitter component demos, animation niche | $80K/month, solo founder | Closest playbook — visual demos of motion |
| Supabase | Launch weeks (1 feature/day for 7 days, every 3–4 months) | 81K stars, 4.5M devs | Predictable cadence > random launches |
| Arc Browser | Invite-only to 50 taste-makers | Massive pre-launch hype | Seed to 20–50 right people > 10K random users |
| Stripe | Documentation as marketing | $95B+ valuation | Great docs = distribution channel |

---

## Relevance to Kvalt

The GTM strategy should mirror the product philosophy: opinionated, not generic. This means:
1. One exceptional visual demo per week, not daily content volume
2. AI-native distribution (configs AI tools consume) as the structural advantage
3. Product quality as the primary marketing channel
4. Free open-source faces as top-of-funnel, GitHub as distribution
5. Community embedding over community building
6. Audience building starts now, not at product launch
