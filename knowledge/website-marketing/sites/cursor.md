# Cursor.com — Website Inspiration Analysis

**URL:** https://cursor.com
**Added:** 2026-04-08
**Category:** AI code editor (developer tool)

---

## Hero Strategy

**Headline:** "Built to make you extraordinarily productive, Cursor is the best way to code with AI."
— Long-form, declarative, single sentence. Reads like a thesis statement, not a tagline.

**CTA:** "Download for macOS" — single CTA, platform-detected. No "Learn more" fallback. Maximum-confidence conversion.

**Visual:** Full interactive demo of the IDE — not a screenshot. Shows simulated task board, browser preview, and chat input with placeholder "Plan, search, build anything..."

### Kvalt application
- Interactive demo-as-hero: embed a live component playground, not a screenshot.
- Single platform-aware CTA: "Install for React" or "npx create-kvalt".
- Placeholder text communicates capabilities subtly.

---

## Product Explanation

**Four feature sections**, identical pattern:
- Left: h3 heading + one sentence + arrow link
- Right: fully built-out HTML/CSS product simulation (not screenshots)
- Entire card is a clickable link — large targets, no separate CTA buttons

**Demo quality:** Slack section shows real messages. GitHub section shows actual PR diffs with bug detection. Built-out recreations, not static images.

**"Acme Labs" fictional company:** Self-referential — fictional company description mirrors Cursor's identity. Meta effect.

### Kvalt application
- Live component demos in feature sections. Live Button with states, live DataTable with sorting.
- Full-card-as-link pattern reduces button clutter.
- Self-referential: use Kvalt components to build the Kvalt site, make that visible.

---

## Visual Design

- **Dark mode first.** System/Light/Dark toggle in footer (not header).
- **Period-terminated headings:** "The new way to build software." — statement of fact, not excitement.
- **Restrained color:** Neutral/dark. Product demos provide the color.
- **Typography:** Large confident headings.

### Kvalt application
- Periods after headings for declarative confidence.
- Dark mode first for developer audience.
- Let component demos carry visual interest.

---

## Navigation / IA

- **4 top-level items:** Product, Enterprise, Pricing, Resources + Sign in/Download.
- **Footer:** 5 columns (Product, Resources, Company, Legal, Connect).
- **External link indicator:** "↗" for outbound links.

### Kvalt application
- 4-5 nav items max. Footer is the sitemap.

---

## Copy / Messaging

- **Declarative, present-tense.** Avoids superlatives.
- **Sentence fragments as headings:** "Agents turn ideas into code" / "Magically accurate autocomplete"
- **Benefit-first:** "Accelerate development by handing off tasks"
- **Research identity:** "Cursor is an applied research team" — researchers, not just builders.
- **Recruiting as brand:** "Join us" mid-page doubles as brand statement.

### Kvalt application
- "Design engineering research" > "component library."
- Benefit-first copy: "Ship consistent UIs by installing one design language."

## Unique Patterns

1. **Product-as-demo, not product-as-screenshot** — every feature section is a fully built-out simulation.
2. **Changelog on homepage** — 4 recent updates with dates. Signals velocity.
3. **Blog as "Recent highlights"** — editorial curation framing.
4. **Closing CTA bookend:** "Try Cursor now." — 3 words, period. Same button as hero.
5. **Questions UI in agent demo** — collaborative AI, numbered options + Skip/Continue.
6. **Theme toggle in footer** — System/Light/Dark with emoji icons.
