## Core Concept

AI is introducing a new interaction paradigm: **intent-based outcome specification**. Instead of telling computers *how* to do something step by step (search flights, then hotels, then activities, manually compare), users describe *what outcome they want*.

**Outcome-oriented design** = orchestrating experience design with greater focus on user goals and final outcomes, while strategically automating aspects of interaction and interface design.

## The Shift

**From:** Designing for the average — perfect search filters, carefully laid out results pages, optimized components for the majority.

**To:** Designing for the individual — identifying different sets of requirements the system operates within while helping each user achieve their specific goals.

## Key Metaphor

Instead of designing a single rigid path through a forest, designers define the **boundaries of where paths can go** and **what makes a good path**. AI then creates specific paths based on each traveler's needs.

## How the Designer Role Evolves

- From designing static interfaces → becoming **architects of possibilities**
- From creating a single experience → designing **frameworks that adapt to individual needs**
- From step-by-step interaction design → **orchestrating adaptive experiences**
- Role becomes more **strategic** — focusing on outcomes that matter to users

## Skills That Remain Crucial

- User-centric problem-solving
- Critical thinking
- Holistic thinking

NNgroup's position: "Our job isn't disappearing, it's evolving."

## Relevance to Kvalt

- **Kvalt faces are exactly this pattern.** A face defines the boundaries (tokens, motion rules, component behaviors, accessibility patterns) — Claude Code creates specific implementations within those boundaries. The face is the "what makes a good path" definition.

- **Validates the spec export direction.** The spec schema we're building for David's handoff is literally a formalized version of "outcome specification" — describing the desired outcome (design intent) rather than the step-by-step implementation (shadcn code).

- **"Designing for the individual" maps to face theming.** The planned theming tool (change DS parameters per client) is outcome-oriented design applied to design systems — same framework, different paths per brand.

- **Strengthens the "design intelligence layer" positioning.** If NNgroup is saying designers become "architects of possibilities" who define frameworks, Kvalt is a tool for encoding those frameworks in a way AI can execute on.

- **Counter to the "AI replaces designers" narrative.** This supports Jan's position that domain expertise (design taste, system thinking) is the moat — AI handles execution, humans define what "good" means.
